div and span elements

div is a division, a section of your page. It has no meaning and no default style.


span is similarly meaningless, but it is has "inline" style instead of "block" style.

This p tag is inside a div, not that we could tell. The div just organizes our page, seperating it out into sections. This text is inside a span element, inside the p element

Inline vs. block

Block elements "own their own vertical space". They take up a horizontal band across the screen. Tables, h1-6s, p tags, div tags, are all block elements.
Inline elements don't own their own vertical space -- they make use of the space controlled by their parent element. A tags and span tags are inline elements.
The above is default behaviour -- we can change it with styling.

We can also say that div is the generic block element and span is the generic inline element

Additional Inline Elements

Since inline elements just inherit space and fit in the same line, they can be used to mark text for decoration. Emphasize with em, u for underlined, but don't use this (see MDN). Deprecated., Strong for something bold-like, doesn't have to be bold.

Typical Use of div and span

Since div and span are generic, we typically assign them class(es) attributes to specify their meaning and style. To some extent, div has been replaced with semantic tags in HTML5.

Semantic Tags (new in HTML5!)

Semantic tags were built with the intent of replacing many divs. Your semantic tags have no default style and represent structure, but each has a name with a meaning.
